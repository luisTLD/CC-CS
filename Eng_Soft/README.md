# Projeto de Software - Fundamentos e Práticas

## Fundamentos

- **Introdução a Projeto de Software** - Conceitos básicos, ciclo de vida e importância do planejamento no desenvolvimento.

- **Qualidade de Software** - Princípios e métricas para garantir sistemas confiáveis e eficientes.

- **Requisitos de Software** - Levantamento, análise e documentação das necessidades do sistema.

## Modelagem com UML

### Modelagem Estrutural
- **Diagramas Estruturais (UML)** - Representação da estrutura do sistema, como classes e relacionamentos.

### Modelagem Comportamental
- **Diagramas Comportamentais (UML)** - Representação das interações e fluxos do sistema ao longo do tempo.

## Arquitetura e Design

- **Arquitetura de Software** - Organização de alto nível dos componentes e suas interações.

- **Camada de Domínio** - Modelagem das regras de negócio e entidades principais do sistema.

- **Persistência de Dados** - Técnicas para armazenamento e recuperação de dados.

## Padrões e Reuso

- **Padrões de Projeto (Design Patterns)** - Soluções reutilizáveis para problemas comuns de design.

- **Padrões Arquiteturais** - Estruturas amplas para organização de sistemas (ex: MVC, camadas).

- **Frameworks e Reuso** - Uso de bibliotecas e estruturas prontas para acelerar o desenvolvimento.
